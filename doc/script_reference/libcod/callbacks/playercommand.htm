Player Command

Module: Callbacks

Summary:

Calls CodeCallback_PlayerCommand() in maps/mp/gametypes/_callbacksetup.gsc when a client issues a command (e.g., a chat message). Requires self processClientCommand to be executed for the command to be processed as usual. Valid commands are:

  • say
  • say_team
  • score
  • mr
  • give
  • take
  • god
  • demigod
  • notarget
  • noclip
  • ufo
  • kill
  • follownext
  • followprev
  • where
  • callvote
  • vote
  • setviewpos
  • entitycount
  • printentities

Note: The existance of the callback function is checked only once on server start. Functions added later are not taken into account, even after a map switch.

Note: Avoid using wait statements (unless threaded) in this callback before calling processClientCommand.



Called on:

The player



Parameters:

  • 1 : <args> Array of strings that constructs the command sent by the client.


Example:

CodeCallback_PlayerCommand(args)
{
    output = "";
    for(i = 0; i < args.size; i++) {
        output += args[i] + ", ";
    }
    self iprintLnBold("command: " + output);
    self processClientCommand();
}
